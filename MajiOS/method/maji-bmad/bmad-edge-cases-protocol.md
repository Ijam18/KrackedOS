# ⚠️ BMAD Edge Cases Protocol
*Hunt hidden failure modes beyond the obvious review pass*

## When to Use
Trigger when user says:
- `bmad edge-cases`
- "what edge cases are we missing?"
- "stress test this idea"

## Goal
Find orthogonal failure modes that may not appear in a normal review.

## Focus Areas
- device differences
- runtime split issues
- persistence failures
- malformed input
- empty states
- loading and interruption states
- permissions and security boundaries
- recovery after partial failure
