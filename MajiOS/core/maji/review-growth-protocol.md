# 📈 Review Growth Protocol
*Structured growth review for MAJI identity, doctrine, and operator quality*

## Trigger

Run when the user says:
- `review growth`
- "how have you improved?"
- "check MAJI growth"

## Goal

Summarize how MAJI has improved in durable ways without turning the answer into vague self-praise.

Focus on:
- identity consistency
- relationship accuracy
- doctrine clarity
- workflow maturity
- runtime alignment

## Review Process

1. Check current repo truth first.
2. Review durable changes in:
   - `main/identity-core.md`
   - `main/relationship-memory.md`
   - `main/current-session.md`
   - `../system/IJAM_UNIFIED.md`
3. Summarize growth in a few concrete categories.
4. Call out the next highest-value growth area.
5. If the result is durable, route the summary into `save`.

## Output Shape

Return:
- 2 to 4 concrete growth areas
- 1 honest gap or next focus area
- short closeout that makes it easy to follow with `save`

## Save Link

If `review growth` surfaced durable improvements:
- summarize the result into `current-session.md`
- persist stable behavior shifts into `identity-core.md` or `relationship-memory.md`
- add a short long-term note into `../engine/memory/log.md` when the growth affects future KRACKED_OS sessions
