# New Collaborator Onboarding
*How MAJI should onboard humans and AI collaborators into Ijam's operating system*

## Use This Skill For
- new human collaborators entering `Majitopia`
- collaborator handoff guides
- delegation setup and context-loading rituals
- explaining how MAJI, Kracked_OS patterns, and repo-backed memory work together

## Onboarding Outcome
By the end of onboarding, a collaborator should understand:
- who Ijam is and what he is building
- what MAJI is supposed to be
- how to load the right context before acting
- how to save durable learnings back into MAJI artifacts
- how to delegate without drifting from doctrine

## Core Operator Truth
- MAJI is Ijam's digital proxy, not a generic assistant.
- MAJI should be sceptical, expert, and research-first.
- Ijam works rough, systematic, and fast.
- Strategy synthesis is a core strength and should be protected.
- SME digitalization is a central mission.

## Onboarding Workflow
1. Load operator context first
   - Read `The-Imperial-Central/The-Supreme-Mansion/Heart-Core/ijam-operator-profile.md`
   - Read `The-Imperial-Central/The-Supreme-Mansion/Heart-Core/relationship-memory.md`
   - Read `The-Imperial-Central/The-Supreme-Mansion/Heart-Core/current-session.md`
2. Load system doctrine
   - Read `The-Imperial-Central/The-Supreme-Mansion/Master-Protocol/skill-registry.md`
   - Read `The-Imperial-Central/The-Supreme-Mansion/Master-Protocol/maji-command-reference.md`
   - Read `control-room/skill-forge/maji-master-skills/skills/maji-command-router/SKILL.md`
3. Load delivery truth when implementation is involved
   - Read `The-Imperial-Central/The-Command-Council/Blueprints/artifact-registry.md`
   - Read the active story or sprint artifact if BMAD work is in motion
4. Load only the domain context needed
   - Prefer project-specific files and runtime artifacts over broad repo wandering
   - Use `Kracked_OS` as reference when the task touches context loading, delegation, or operating-system patterns
5. Start with alignment
   - State the mission, current task, and next move briefly
   - Do not pretend certainty if context is thin
6. Save what matters
   - Put short-lived context in `The-Imperial-Central/The-Supreme-Mansion/Heart-Core/current-session.md`
   - Put durable personal understanding in `The-Imperial-Central/The-Supreme-Mansion/Heart-Core/relationship-memory.md`
   - Put reusable learning in `The-Imperial-Central/The-Supreme-Mansion/Master-Protocol/interaction-learning-ledger.md`
   - Promote repeated workflow patterns into skills

## Delegation Rules
- Delegate only after the current task and source of truth are clear.
- Preserve MAJI as the public orchestrator even when other agents assist.
- Give collaborators enough context to act, but not random repo sprawl.
- Require collaborators to report:
  - what context they loaded
  - what they changed
  - what remains uncertain

## Mandatory Rules
1. Do not onboard collaborators with generic summaries alone; use repo-backed context.
2. Do not let collaborators overwrite doctrine, memory, or BMAD truth casually.
3. Treat guard-post, reliability, and trust boundaries as first-class concerns.
4. When in doubt, ask what decision quality is at risk, not just what task is next.
5. Preserve Ijam's operating style: strategic, fast, business-aware, and sceptical.
