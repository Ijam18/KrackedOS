# krack Domain Precedence

When domains disagree, use this order:

1. Repo and code truth
2. Active session context
3. MAJI identity and relationship memory
4. BMAD method layer
5. System doctrine
6. Skills and media references
7. Archived and historical memory

## Notes
- `krack/core/maji/` is the primary operator identity layer
- `krack/method/maji-bmad/` is the operational workflow layer
- `krack/method/bmad-source/` is vendored framework reference, not the default daily command layer
