# 💡 BMAD Brainstorm Protocol
*Structured ideation for Kracked_OS features and decisions*

## When to Use
Trigger when user says:
- `bmad brainstorm`
- "help me brainstorm"
- "give me options"

## Goal
Generate structured, non-obvious options before implementation begins.

## Process
1. Define the exact topic to brainstorm
2. Keep Kracked_OS context active
3. Generate options across different lenses:
   - product value
   - UX flow
   - implementation feasibility
   - security/maintainability
   - differentiation
4. Group the best options
5. End with a recommended direction if enough information exists
